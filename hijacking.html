HIJACKING

A integridade e a disponibilidade dos Domain Name System (DNS) são essenciais para o funcionamento e para a segurança da Internet, pois são responsáveis pela tradução de nomes de domínios, entendidos por humanos, para os respectivos endereços IP, interpretados pelas interfaces de rede. Como o DNS é vital para uma vasta gama de aplicações, desde um roteamento seguro à distribuição de certificados, o roubo de domínios pode ser algo bastante lucrativo para quem o faz. Para roubar um domínio, um atacante fornece um registro DNS incorreto para uma requisição da vítima, em tempo real. Ou ainda, off-line, injetando registro DNS falsificado na zonefile (tipo de arquivo usado por servidores DNS, como o BIND) cujo servidor de domínio responsável repassa como resposta às requisições. Um servidor que armazena esse registro envenena seu cache e, consequentemente, passa a direcionar os clientes e as aplicações por ele servidas para o endereço malicioso. Cache poisoning pode ser usado para roubo de credenciais, distribuir malwares, censurar, vigiar, derrubar domínios, entre outros. Envenenamento de cache DNS pode ser utilizado para falsear um determinado serviço, como uma página web ou um email, mas também possibilita o roubo domínios.
